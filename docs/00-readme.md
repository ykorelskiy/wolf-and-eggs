# Wolf-and-Eggs — проект-документация

Веб-ремейк советской игры «Ну, погоди!» (Электроника ИМ-02).

## Точка входа для Cline

Каждая сессия начинается с прочтения:
1. Этот файл
2. `07-progress-log.md` — что было сделано в прошлый раз
3. `06-roadmap.md` — текущая фаза

## Структура документации

| Файл | Что внутри |
|------|-----------|
| `01-GDD.md` | Геймдизайн: что делаем и как играется |
| `02-architecture.md` | Модули, контракты, поток данных |
| `03-art-direction.md` | Визуальный стиль, палитра, правила |

## Роли

- **Архитектор / Creative Director** — пользователь (ты)
- **Junior Developer** — Cline в VS Code
- **Outsourced Artist** — Nano Banana
- **Outsourced Sound Designer** — AI sound generator
- **Консультант по архитектуре** — внешний LLM (Claude)

## Принципы

1. Сначала прототип на плейсхолдерах, потом графика
2. Phase не закрыта, пока все критерии приёмки не выполнены
3. Git-коммит каждые 20-30 минут или после завершённой задачи
4. Решения фиксируются в ADR до того, как код написан

# Game Design Document

## Концепция
Веб-ремейк «Ну, погоди!» (Электроника ИМ-02, 1984). Волк ловит 
яйца, скатывающиеся с четырёх жёлобов. Цель — продержаться как 
можно дольше и набрать рекордный счёт.

## Core Loop
1. Игрок переключает позицию волка между 4 точками
2. Яйца спавнятся на 4 жёлобах и катятся вниз
3. Волк должен быть в правильной позиции в момент достижения 
   яйцом края жёлоба → +1 очко
4. Промах → -1 жизнь, разбитое яйцо
5. При 0 жизней → Game Over

## Управление

Волк всегда находится в одной из 4 позиций. Управление — прямое: одна клавиша = одна позиция. Никаких «верх/низ + лево/право» по отдельности.

### Десктоп

| Позиция | Основная клавиша | Дубликат |
|---|---|---|
| Верх-лево (TL)  | `A` | `1` |
| Низ-лево (BL)   | `Z` | `2` |
| Верх-право (TR) | `K` | `9` |
| Низ-право (BR)  | `M` | `0` |

Раскладка симметричная: две клавиши под левой рукой (`A`/`Z`), две под правой (`K`/`M`). Это повторяет ощущение LCD-игр серии «Электроника» и не требует комбинаций.

Служебные клавиши:
- `Enter` / `Space` — старт игры, рестарт после game over
- `P` / `Escape` — пауза / возврат в игру
- Mute — только кнопкой в UI (без горячей клавиши, чтобы не конфликтовать с `M`)

### Мобильный / тач

Экран делится на 4 четверти, тап в четверть переводит волка в соответствующую позицию. Жестов и свайпов нет — только тап.

### Геймпад

Не входит в MVP. Возможный кандидат на Phase 9.

## Жёлоба
4 жёлоба, по одному к каждой позиции волка. Яйцо спавнится в 
верхней точке жёлоба и катится к нижней (углу позиции волка).

## Время скатывания яйца
Базовое время: **3.0 секунды** от спавна до края жёлоба.
Время уменьшается по кривой сложности (см. ниже).

## Кривая сложности

Источник истины — реальные ощущения от оригинала.

| Счёт | Время скатывания | Темп спавна |
|------|------------------|-------------|
| 0    | 3.0с             | 1 яйцо / 2.5с |
| 25   | 2.5с             | 1 яйцо / 2.0с |
| 50   | 2.0с             | 1 яйцо / 1.7с |
| 75   | 1.6с             | 1 яйцо / 1.4с |
| 100  | 1.3с (пик)       | 1 яйцо / 1.2с |
| 101  | 1.6с (откат!)    | 1 яйцо / 1.5с |
| 150  | 1.3с             | 1 яйцо / 1.2с |
| 200  | 1.0с (пик 2)     | 1 яйцо / 1.0с |
| 201  | 1.4с (откат)     | 1 яйцо / 1.3с |
| 300+ | 0.8с (макс)      | 1 яйцо / 0.9с |

Значения — стартовая точка для тюнинга. Финальные цифры 
подбираются playtesting'ом.

### Откат после смерти
При потере жизни:
- Текущая скорость падает на **30%** (длительность скатывания 
  увеличивается на 30%)
- За **10 секунд** плавно возвращается к скорости, 
  соответствующей текущему счёту по таблице выше

## Жизни
- Стартовое количество: **3**
- При промахе (яйцо упало мимо корзины): -1 жизнь
- При 0 жизней: Game Over
- В оригинале была механика «прощения» 2 разбитых яиц при 
  условии что курица спит. У нас в MVP — нет, только классический 
  счёт жизней. Можно добавить в Phase 5.

## Спавн яиц
- Жёлоб выбирается случайно из 4
- Минимальный интервал между спавнами — см. таблицу
- Максимум одновременно на экране: 4 (по одному на жёлоб)
- Если на жёлобе уже катится яйцо — спавн отменяется, выбирается 
  другой жёлоб

## Состояния игры (StateManager)
- `MENU` — стартовый экран
- `PLAYING` — игровой процесс
- `PAUSED` — пауза (Esc / тап по паузе)
- `GAME_OVER` — экран финала со счётом и рекордом
- Переход MENU → PLAYING → (PAUSED) → GAME_OVER → MENU

## Подсчёт очков
- Поймал яйцо: +1
- Промах: 0 очков, -1 жизнь
- Бонусов в MVP нет

## Рекорд
- Сохраняется в `localStorage`
- Показывается на экране MENU и GAME_OVER

## Технические параметры
- **Logical resolution: 320×240** — координатная система игровой логики.
- **Render resolution: 640×480** — размер canvas (×2 upscale от logical).
- Aspect: 4:3
- Upscale через CSS, `imageSmoothingEnabled = false`
- Target FPS: 60
- Pixel-perfect rendering
